# 视频分类脚本使用说明

## 概述

本脚本用于将视频文件按"开头"进行分组，确保每个目标文件夹内的视频"开头"全部不相同。

### 分组规则

- **文件名格式**：`{剧名}开头{编号}_xxxxxxx.mp4`
- **开头相同判定**：剧名 + 开头编号 相同，即为相同开头
  - 例如：`三万英尺开头1_xxx.mp4` 和 `三万英尺开头1_yyy.mp4` → 开头相同
  - 例如：`三万英尺开头1_xxx.mp4` 和 `三万英尺开头2_xxx.mp4` → 开头不同

### 分配原则

- 每个目标文件夹中，同一个"开头"的视频只出现一次
- 脚本采用贪心算法，尽量将不同开头的视频放入同一文件夹，减少文件夹总数

### 文件名兼容性

支持两种格式：
- **有剧名**：`三万英尺开头1_三万英尺 1-5·06_part_006.mp4`
- **无剧名**：`开头8_视频2_part_007_merged.mp4`

## 使用方法

### 环境要求

- Python 3.6 或更高版本（推荐使用 Python 3.x）
- 无需额外安装第三方库
- 支持 Windows / Mac / Linux 系统

### 方式一：Windows 命令行

#### 预览分配结果

```bash
python 视频分类脚本.py "C:\你的视频目录路径"
```

#### 执行移动

```bash
python 视频分类脚本.py "C:\你的视频目录路径" -e
```

执行时会提示确认，输入 `y` 确认执行。

---

### 方式二：Mac 双击启动（推荐）

将以下文件放到**视频所在文件夹**下：

| 文件 | 说明 |
|------|------|
| `视频分类脚本.py` | Python 脚本主程序 |
| `预览.command` | 双击运行，只预览不移动 |
| `执行.command` | 双击运行，执行移动 |

双击对应的 `.command` 文件即可运行。

---

### 方式三：Windows 双击启动（推荐）

将以下文件放到**视频所在文件夹**下：

| 文件 | 说明 |
|------|------|
| `视频分类脚本.py` | Python 脚本主程序 |
| `预览.bat` | 双击运行，只预览不移动 |
| `执行.bat` | 双击运行，执行移动 |

双击对应的 `.bat` 文件即可运行。

---

### 方式四：Mac / Linux 命令行

```bash
python3 视频分类脚本.py "/你的视频目录路径"
```

## 参数说明

| 参数 | 简写 | 必填 | 说明 |
|------|------|------|------|
| `source_dir` | - | 是 | 视频所在的源目录路径 |
| `--execute` | `-e` | 否 | 添加此参数才会执行移动，否则只预览 |
| `--extensions` | `-ext` | 否 | 支持的视频格式，默认为 `mp4,avi,mkv,mov,wmv,flv,webm` |
| `--max-files` | `-m` | 否 | 每个文件夹最多允许放入的文件数量，默认为 25 |

## 常见问题

### 1. Python 版本提示 "编码错误"

在脚本开头添加编码声明（已包含）：

```python
# -*- coding: utf-8 -*-
```

### 2. 未找到符合规则的文件

确保文件名符合格式：`剧名开头数字_xxx.mp4` 或 `开头数字_xxx.mp4`

示例：
- ✅ `三万英尺开头1_三万英尺 1-5·06_part_006.mp4`
- ✅ `开头8_视频2_part_007_merged.mp4`
- ❌ `第一集.mp4`（缺少开头编号）

### 3. 目标文件夹已存在同名文件

脚本会自动跳过已存在的文件，不会覆盖。

### 4. Mac 双击 .command 文件没反应

确保文件有执行权限。如果终端提示 permission denied，在终端中运行：

```bash
chmod +x 预览.command
chmod +x 执行.command
```
