# 项目全景上下文

## 项目背景

本项目源于视频素材整理的实际需求。用户拥有大量混剪视频文件，需要按特定规则进行分类整理。

## 业务背景

### 原始场景

- 用户在 Windows 电脑上管理视频素材
- 视频文件命名格式：`{剧名}开头{编号}_xxxxxxx.mp4`
- 存在大量不同"开头"的视频需要分类

### 业务需求

将视频文件分配到不同文件夹，确保每个文件夹内的视频"开头"全部不相同。

**"开头"的定义**：
- 格式：`{剧名}开头{编号}` 或 `开头{编号}`
- 示例：`三万英尺开头1`、`开头8`

### 核心约束

1. **开头不重复**：同一文件夹内，不能有两个视频的"开头"相同
2. **尽量少文件夹**：在满足约束的前提下，减少目标文件夹数量
3. **预览确认**：执行前必须先预览分配结果，确认无误后再执行

## 技术背景

### 技术选型

- **语言**：Python 3（跨平台，无需安装额外库）
- **部署方式**：将脚本放到视频目录双击运行

### 关键设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 脚本语言 | Python | 跨平台，标准库完整，无需安装 |
| 分配策略 | 贪心算法 | 实现简单，效果满足需求 |
| 交互方式 | 预览 + 执行分离 | 允许用户先确认再操作 |
| 部署方式 | 脚本放视频目录 | 简化使用，无需配置路径 |

## 项目结构

```
视频目录/
├── 视频分类脚本.py       # 主程序
├── 预览.command          # Mac 预览启动（双击）
├── 执行.command          # Mac 执行启动（双击）
├── 视频文件1.mp4
├── 视频文件2.mp4
└── ...
```

## 关键文件说明

| 文件 | 说明 |
|------|------|
| `视频分类脚本.py` | Python 主脚本，支持 Windows/Mac/Linux |
| `预览.command` | Mac 双击启动文件，只预览不移动 |
| `执行.command` | Mac 双击启动文件，执行移动 |
| `使用说明.md` | 详细使用文档 |
| `README.md` | 项目简要说明 |
| `项目全景上下文.md` | 本文档 |

## 使用流程

```
1. 放置文件
   └── 将脚本和启动文件放到视频目录

2. 预览分配
   ├── Windows: python 视频分类脚本.py "目录"
   └── Mac: 双击 "预览.command"

3. 确认结果
   └── 检查预览输出是否合理

4. 执行移动
   ├── Windows: python 视频分类脚本.py "目录" -e
   └── Mac: 双击 "执行.command"
```
